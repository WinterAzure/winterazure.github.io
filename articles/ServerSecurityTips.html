[测试1-SSH密钥部署]

[目标]

  • 增强服务器安全性，关闭密码验证，使用密钥验证
  • 更改默认ssh端口，防火墙配套设定

[步骤]

  1. ssh密码登录目标主机

  2. 使用ssh-keygen在本机上生成密钥对。KDF迭代次数至少为256，密钥类型应当为ed25519。使用默认的保存位置，设置私钥口令。 私钥口令不应当与用户密码相同。

    > ssh-keygen -o -a 256 -t ed25519

  3. 使用ssh-copy-id上传公钥

    > ssh-copy-id user@remote -p port

  4. 更改ssh配置，允许公钥登录,更改默认端口，保留目前的session。 /etc/ssh/sshd_config 中 RSAAuthentication 改为 yes Port 改为非22端口 重启sshd，应用更改.

> sudo systemctl restart sshd

  1. 测试本地使用公钥登录

    > ssh -p NewPort user@server

  2. 更改ssh配置，禁止密码登录，禁止root登录，禁止空密码，激活严格模式，禁止X11转发

  • /etc/ssh/sshd_config 中 PasswordAuthentication 改为 no

  • PermitEmptyPasswords 改为 no

  • PermitRootLogin 改为 no

  • StrictModes 改为 yes

  • X11Forwarding 改为 no

  1. 重启sshd，参见Step5，测试是否可以使用密码登录，应当不能。

  2. 关闭旧的连接，使用公钥重新登录。

  3. 设置ufw，允许出站，拒绝入站，特殊允许ssh端口tcp入站连接

    > sudo ufw default allow incoming
> sudo ufw default deny outgoing
> sudo ufw allow ssh_port/tcp
> sudo ufw enable

  4. 查看ufw状态。确认更改已经生效。

    > sudo ufw status verbose

  5. 测试ssh登录，确认更改生效，ssh可以正常登录。 12 .查看目标主机的网络监听状态，确认只有sshd与systemd-resolve在监听对应的端口

    sudo ss -tlp

[测试2-doas替换测试]

[目标]

  • 使用doas代替sudo，禁用sudo，提高安全性

[步骤]

  1. 在本机克隆并编译doas

    git clone https://github.com/slicer69/doas

  2. 将本机编译完成的doas复制到目标主机

    scp -P ssh_port local_file user@server:/tmp

  3. 将目标文件拷贝到/usr/bin/，删除源文件

  4. 创建doas配置文件，位置为/usr/local/etc/doas.conf

    文件内容：

    permit <username> as root

    4.设置doas的权限，允许其setuid

    sudo chmod 4755 /usr/bin/doas

    5.测试doas是否可以运行 6.设置bash alias，使sudo指向doas

    alias sudo="doas "

[测试3-Nginx证书部署测试]

[目标]

  • 设置nginx证书

[前置条件]

  1. 拥有证书私钥
  2. 已安装nginx
  3. 已经拥有可用的配置文件

[步骤]

  1. 检查nginx是否支持ssl

    > nginx -V  寻找-with-http_ssl_module

  2. 配置文件 /etc/nginx/sites-avaliable/host-name.conf 添加 ssl_certificate和ssl_certificate_key /etc/nginx/sites-enabled/host-name.conf 添加 ssl_certificate和ssl_certificate_key
