OSI 7 层模型与TCP 5层模型

定义：1.网络协议(Network Protocol)：如何发送和接收信息的一套规则 2.接口(Interface)：同一计算机的不同功能层之间的通信规则 3.冲突域(Collision Domain)* ：发送数据给一个单一目标所影响的范围 4.广播域(Broadcast Domain)* ：发送数据给一个不明确目标所影响的范围 5.媒体访问控制/物理地址(Media Access Control/MAC)：用来确定网络设备位置的地址 6.内容可寻址存储器(Content Addressable Memory/CAM)：网桥所保存的动态MAC表，动态刷新

OSI 7层模型(Open System Interconnection/Reference Model, OSI/RM)

  1. 物理层(Physical Layer) 作用：完成相邻结点之间原始比特流的传输,控制数据怎样被放置到通信介质上（如何在电缆上传输 0/1，如何让机器们知道获取这些0/1） 设备：1. 中继器(Repeator):把电缆上的信号放大续传，延长以太网的长度。

    1. 集线器(Hub):多个端口，把任意一个端口传过来的信号发到其他所有端口。

      协议：IEEE802.2，Ethernet以太网，Internetwork

    2. 数据链路层(Data Link Layer) 作用：在网络节点中利用MAC地址进行可靠的数据传输，实现设备间点到点通讯（把封装好的帧 (Frame)顺序传输，添加校验CRC并解决因为帧的损坏导致的问题，进行流量控制防止速度高 的淹没速度低的） 设备：1. 网卡(Network Interface Card,NIC):整理设备要发送的数据，分割成数据包并发送出去。

       网卡特点:每个NIC只有一个唯一的MAC地址
       2. 网桥(Bridge):高级的集线器，利用MAC来判断要转发数据包与否。可以用来分割冲突域。
           网桥的工作**:通过对经过自己的包的源MAC和目的MAC进行记录和更新，判断各个机器分别在什么区域里，决定是否要转发包
       3. 交换机(Switch):多端口的网桥+集线器，每一个端口都对应一个区域，维护CAM，利用MAC来判断转发与否

      协议：ARP，PPTP，L2TP，RARP

    3. 网络层(Network) 作用：利用路由算法，完成网络中主机间的报文传输，包括在广域网中两个不同网络中主机的数据 传输。 设备：1. 路由器(Router):使用IP地址连接设备，连接多个不兼容的网络，构成更大的网络。

        路由器特点: 1. 数据通道功能:由硬件实现转发决定,背板转发,输出链路调度等功能。
            2. 控制功能:由软件实现信息交换、系统配置、系统管理等功能。
            3. 判断网络地址和选择路径:智能地选择最佳路由，多快好省。
            4. 阻止广播，可以内置防火墙，安全性高。

      协议：IP，IPv6，ICMP，ICMPv6

    4. 传输层(Transport Layer) 作用：实现用户进程端到端通讯，作为用户功能的最低层，提供错误处理，拥堵控制功能。 设备：略（大量互联网设备，特别是终端设备） 协议：TCP，UDP，移动IP

    5. 会话层(Session Layer) 作用：允许不同机器上的用户之间建立会话关系，提供令牌控制功能和数据同步功能。 设备：同上 协议：SSL，TSL，RPC

    6. 表示层(Presentation Layer) 作用：把下层传输的比特流转换为含有特定语义和语法的信息，对采用不同的编码方法的，已经转化 为比特流的信息重新还原为由他们的编码方法定义的格式，如ASCII编码。同时提供数据压缩 和解压、数据加密和解密功能 设备：同上 协议：XDP，LPP

    7. 应用层(Application Layer) 作用：用于进行特定应用，实现实用的功能。 设备：同上 协议：HTTP，FTP，STMP，Xwindow

TCP/IP 4层模型(Transmission Control Protocol/Internet Protocol,传输控制协议/网际协议)

OSI模型具有通用性，TCP/IP模型仅适用于TCP/IP协议族。

1. 网络访问层(Network Access)

网络访问层相当于OSI模型中的数据链路层和物理层

2.网际层(Internet)

网际层相当于OSI模型中的网络层，其功能有： 1. 处理传输层的分组发送请求，即处理和传输数据报 2. 处理输入数据报，即检查数据报是否目的为本机，决定转发与否，另有校验功能 3. 处理ICMP报文，提供路由选择，流量控制，拥塞控制功能

3.传输层(Transport)

传输层相当于OSI模型中的传输层

4.应用层(Application)

应用层相当于OSI模型中的会话层，表示层，应用层 ### ICMP协议(Internet Control Message Protocol,Internet 控制消息协议)

  1. 简介：由于IP协议不提供差错控制，检验，重发等功能，对网络故障的报告，传递控制消息(例如网络是否通畅，主机是否可达到)需要用到ICMP协议。ICMP协议用IP模块实现。

  2. 原理：当某个网关发现传输错误时,立即向信源主机发送 ICMP 报文,报告出错信息,让信源主机采取相应处理措施。一个 ICMP 报文包括 IP 头部、ICMP 头部和 ICMP 报文,IP 头部的”协议“值为 1 就说明这是一个 ICMP 报文。

  3. 结构(以下内容封装在IP数据包的数据部分)：

    |Type|Code|checksum|
--------------------
| Rest Of Header   |
--------------------

    解释：1. Type：由于说明ICMP报文作用及格式

     2. Code详细说明某种ICMP报文的类型
    常见的类型*：1. 发送Type=8，响应Type=0：说明主机存在，应用于ping程序
               2. 返回Type=3，Code=3：目标不可达

    另见：* ：CCNA：P40表2-4-1，P41，RCF792，维基百科上的ICMP内容。### IP协议

  4. 简介：TCP/IP网际层把传输层的数据报封装在IP数据包中，再将其发送。IP协议的作用就是把数据传输到目的地。IP协议不负责可靠性，流控制，顺序控制。这些功能由上层协议提供。IP协议提供路由功能，负责选择传输路线。

    主要名词：

        1. 优先级和服务类型：又称服务质量(Quality of Server/QoS)，提供一种控制机制， 为不同的应用和用户提供优先级不同的服务；
    2. 存活期(Time To Live/TTL):数据包可生存的上限，初始值由发送者设置，每经过一次路由，TTL值减一，到0时抛弃。防止数据包在环形网络中无尽传输。
