不同语言中文件操作方式(C,C++)

文件操作是几乎所以程式设计语言都要提供的功能，有点包含在标准库里面，有的自己提供。我在这里记录一下自己学到的不同语言的典型的文件操作方法。虽然学艺不精但还是记录一下免得搞混了XD

C (标准库方法)

C 的标准库里面包含了文件操作的接口，而且是各个平台都支持的。这完全没有必要多说因为几乎只要是上过大学学过C的都会用，而且即使忘记了只需要man stdio就可以了。

C (UNIX 系统调用)

UNIX自己有底层的系统调用。考虑到“一切皆文件”的传统自然文件方面的接口十分重要。大部分讲关于UNIX系统编程的书都有讲它，而且C标准库里面的fopen,fread之类的函数也是对这些底层的二次封装。只要strace一下，就可以看到很多open,close,read.它们出现的是那么早，我觉得这也是为什么C标准库里面要用fopen,fclose,fread:因为名字被人家用完了XD.

为了使用这些系统调用，你需要一个fcntl.h的头，通常也会一起带上unistd.h,sys/types.h之类的。里面大部分都可以在man手册里面找到。下面是一些常用的：

  • int open(Filename,Mode)

    这个函数打开指定的文件(const char *),返回一个int也就是文件描述符。模式是一些宏，而不是像stdio里面的"r","w"什么的。比如只读打开就是O_RDONLY。这些都在手册里面有。

  • int creat(dirname,mode)

    这个函数创建一个全新的文件,mode是以数字表示的权限位，比如0644,0777之类的，同样返回的是一个文件描述符。

  • ssize_t read(int fd, void *buf, size_t count);

    这个函数用来从一个文件描述符里面读取指定大小的数据到一块内存里面去。返回的是读取的字节数。注意返回的和你需求的可能不太一样，因为文件可能已经读完了。

  • ssize_t write(int fd,const void *buffer,size_t count)

    这个函数写数据到文件描述符。返回的同样是写入的字节数。如果失败了就返回-1。

作为一个示例，用这些系统调用可以拼凑出一个简单的cp程序：

#include <fcntl.h>
#include <unistd.h>
#include <stdio.h>

#include <sys/stat.h>

#define BUFFSIZE 8192

int main(int argc,char *argv[]){
    if (argc!=3){
        printf("Bad usage.\n");
        return 1;
    }

    int fd_read,fd_write;
    int number_read,total_copied;
    char buff[BUFFSIZE]={0};
    struct stat fp_info;

    if ((fd_read=open(argv[1],O_RDONLY))==-1){
        perror(argv[1]);
        return 2 ;
    }

    if ((fd_write=creat(argv[2],0644))==0){
        perror(argv[2]);
        return 3;
    }

    if (fstat(fd_read,&fp_info)!=0){
        perror(argv[2]);
        return 4;
    }

    while ((number_read=read(fd_read,buff,BUFFSIZE))!=0){
        total_copied+=number_read;
        printf("\033[A\33[2KT\rTTotal %ld, copied %ld\n",fp_info.st_size,total_copied);
        if (write(fd_write,buff,number_read)!=number_read){
            perror("Write file error.\n");
            return 4;
        }
    }
    close(fd_write);
    close(fd_read);
    return 0;
}

其中那个诡异的 \033[A\33[2KT\rTT 作用就是”清除当前一行的内容，覆盖打印接下来的东西“.

不过，到底是标准库快呢，还是直接用系统调用快呢？我做了一个小实验。buffsize都是4096，对一个1G的文件拷贝，看看C Stdio(glibc),fcntl,fstream哪一个快一点。

小实验

C(fcntl)

#include <fcntl.h>
#include <unistd.h>

#define BUFFSIZE 4096

int main(int argc,char *argv[]){
    int fd_read,fd_write;
    int number_read;
    char buff[BUFFSIZE]={0};

    if ((fd_read=open(argv[1],O_RDONLY))==-1)
        return 2 ;

    if ((fd_write=creat(argv[2],0644))==0)
        return 3;

    while ((number_read=read(fd_read,buff,BUFFSIZE))!=0){
        if (write(fd_write,buff,number_read)!=number_read)
            return 4;
    }
    close(fd_write);
    close(fd_read);
    return 0;
}

编译使用

gcc-11 -Og copy.c -o copy_c

三次测试得到的数值是：

| 0.184s | 0.184s | 0.207s |

C(stdio)

#include <stdio.h>

int main(int argc,char *argv[]){
    FILE *fp_in=fopen(argv[1],"rb");
    FILE *fp_out=fopen(argv[2],"wb");
    char buffer[4096]={0};
    size_t bytes_read=0;

    if (fp_in==NULL||fp_out==NULL)
        return 1;

    while ((bytes_read=fread(buffer,1,4096,fp_in))>0){
        fwrite(buffer,1,bytes_read,fp_out);
    }

    return 0;
}

编译使用

gcc-11 -Og copy_stdio.c -o copy_stdio

三次测试结果：

| 0.217s | 0.218s | 0.199s |

C++(fstream)

#include <fstream>

using namespace std;

int main(int argc,char *argv[]){
    ifstream ifs;
    ofstream ofs;
    char buffer[4096]={0};

    ifs.open(argv[1]);
    ofs.open(argv[2]);

    if (!ifs.is_open()||!ofs.is_open())
        return 1;

    while (!ifs.eof()){
        ifs.read(buffer,4096);
        ofs.write(buffer,ifs.gcount());
    }

    return 0;
}

编译使用

g++ -Og copy_fstream.cxx -o copy_fstream

三次测试结果：

| 0.212s | 0.216s | 0.228s |

C++(filesystem)

#include <filesystem>

namespace fs=std::filesystem;

int main(int argc,char *argv[]){
    fs::copy_file(argv[1],argv[2]);
    return 0;
}

编译使用

g++ -Og -std=c++17 copy_fs.cxx -o copy_fs

三次测试如下：

| 0.127s | 0.131s | 0.129s |
