在Windows+cygwin平台上编译fltk

目标很明确，就是要编译安装fltk工具包。

注意事项

  1. 按照这个流程编译出来以就需要cygwin1.dll

  2. 不能把cygwin的libuuid-devel恢复原样。

1.下载fltk源码包

在官方下载站点下载最新的fltk源码包。现在最新的是版本1.3.8，解压。

2.配置configure脚本

当然你需要已经安装好了编译器套件和autoconf流。不过这里不能简单的./configure && make -j8 && make install，而是需要额外的配置CC,CXX和CPP。

注意这里使用的方法依旧是最后编译出来的程序需要cygwin1.dll的。

 CC=x86_64-w64-mingw32-gcc CXX=x86_64-w64-mingw32-g++ CPP=x86_64-w64-mingw32-cpp

接下来

./configure --enable-shared

即可。 不出意外的话，结果应该是类似

3.编译

直接make -j8即可。（当然并行的数量取决于你的CPU） 然后，不出意外，遇到第一个报错。重新单线程make一下，确定错误：

make[1]: Entering directory '/home/windows-user/fltk-1.3.8/src'
g++ -L. -mwindows -Wl,-no-undefined -Wl,-Bsymbolic-functions -Wl,-gc-sections -shared -Wl,--whole-archive -Wl,--export-all-symbols -Wl,--enable-runtime-pseudo-reloc -Wl,--enable-auto-import -Wl,--enable-auto-image-base -o mgwfltknox-1.3.dll ../lib/libfltk.a ...
......
/usr/lib/gcc/x86_64-pc-cygwin/11/../../../../x86_64-pc-cygwin/bin/ld: ../lib/libfltk.a(fl_dnd.o):fl_dnd.cxx:(.rdata$.refptr.IID_IUnknown[.refptr.IID_IUnknown]+0x0): undefined reference to `IID_IUnknown'
collect2: error: ld returned 1 exit status
make[1]: *** [Makefile:442: mgwfltknox-1.3.dll] Error 1
make[1]: Leaving directory '/home/windows-user/fltk-1.3.8/src'
make: *** [Makefile:22: all] Error 1

这是一个正常现象。原因出在cygwin的libuuid-devel库和windows的UUID库冲突。fltk配置使用的是windows的库，找到的却是cygwin的库，自然函数接口不一样。解决方法很简单，就是把C:\%cygwin安装路径%\lib\libuuid.dll.a先挪走，确保ld找到的是windows的库即可。

4.继续编译

解决这个问题以后继续make -j8，不出意外的话收获第二个报错：

ExternalCodeEditor_WIN32.cxx: In member function ‘int ExternalCodeEditor::start_editor(const char*, const char*)’:
ExternalCodeEditor_WIN32.cxx:424:3: error: ‘_snprintf’ was not declared in this scope; did you mean ‘vsnprintf’?
  424 |   _snprintf(cmd, sizeof(cmd), "%s %s", editor_cmd, filename);
      |   ^~~~~~~~~
      |   vsnprintf

这个问题不知道是为什么，不过解决的方法很简单，按照编译器的提示替换就可以了。在源码根目录一行perl下去，问题解决。

perl -pe 's/_snprintf/vsnprintf/g' fluid/ExternalCodeEditor_WIN32.cxx > fluid/ExternalCodeEditor_WIN32.cxx

5.再次继续编译

接下来是第三个报错：

Linking fluid-shared.exe...
......
/usr/lib/gcc/x86_64-pc-cygwin/11/../../../../x86_64-pc-cygwin/bin/ld: fluid.o:fluid.cxx:(.text.startup.main+0x26b): undefined reference to `ExternalCodeEditor::set_update_timer_callback(void (*)(void*))'
collect2: error: ld returned 1 exit status
make[1]: *** [Makefile:61: fluid-shared.exe] Error 1
make[1]: Leaving directory '/home/windows-user/fltk-1.3.8/fluid'
make: *** [Makefile:22: all] Error 1

是关于一个fltk的UI设计辅助程序fluid的链接错误。但其实这个程序对库来讲无关紧要，所以直接忽略即可。 那么，到这里fltk自身已经编译完毕了，直接make install就行了，如果最后的结果是出错，而且出错信息和上面的一样，那么就是安装成功了。

6.测试

一个简单的测试

#include <FL/Fl.H>
#include <FL/Fl_Window.H>
#include <FL/Fl_Box.H>

int main(int argc, char **argv){
  Fl_Window *window = new Fl_Window(340, 180);
  Fl_Box *box = new Fl_Box(20, 40, 300, 100, "Hello, World!");
  box->box(FL_UP_BOX);
  window->end();
  window->show(argc, argv);
  return Fl::run();
}
